Display and Positioning

Block Elements

Divs are block elements (as are paragraphs, headings, lists and list items)
By default, they take up 100% of the available width
And they stack on top of each other

Inline Elements

Here are some inline elements. They do NOT take up the full width and width and height properties are ignored.

A span is useful for targeting some text. These ones are strong (bolded) and em (italic). A link, like this one to Google is also inline.

Open the developer tools and see what happens when you make these elements inline-block or block elements.

Images

Images are weird. They are inline, but they do have a height and width unlike spans and links and other inline elements. Why???

Only set the width or the height, but not both to maintain the aspect ratio

Position

A sticky box will hover above the content. Try using position absolute instead.

Position can remove an element from the "flow" of the document. Use with caution!
